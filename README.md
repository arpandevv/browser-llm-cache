# 🤖 browser-llm-cache

> Low-RAM, streaming, hash-verified cache and pre-fetcher for large model weights (LLMs, Embeddings, Diffusion models) running client-side.

Modern browsers can run AI models locally using WebGPU, Web Assembly, and libraries like `@huggingface/transformers` or `web-llm`. However, downloading gigabytes of weight files (like `.safetensors` or `.onnx`) poses significant challenges:
1. **Memory Overload:** Downloading a 2GB file directly into JavaScript RAM can crash browser tabs (especially on mobile).
2. **Slow Checks:** Verifying the SHA-256 hash of cached gigabyte-scale files on every reload can freeze the main thread for seconds.
3. **Complex Progress Tracking:** Displaying real-time download speed, progress percentages, and estimating completion times for multiple assets concurrently is difficult.

`browser-llm-cache` solves these issues using a memory-constant **Streams API** interceptor, **Cache Storage API** for streaming directly to disk, and **IndexedDB** for caching verified file metadata.

---

## Features

*   📦 **Memory-Constant Caching:** Streams downloads directly to the browser's Cache Storage disk, keeping JS heap memory flat regardless of file size.
*   ⚡ **Instant Cache Verification:** Verifies the cryptographic hash (`SHA-256`) during the initial download stream. Subsequent loads verify against an IndexedDB metadata store, making cache hits instant.
*   🎣 **React Hooks Integration:** Track concurrent multi-file downloads with overall progress, file-by-file progress, speed tracking (bytes/sec), error states, and cancellation support.
*   🔌 **Library Agnostic / Transformers.js Ready:** Expose a custom `fetch` override that integrates directly into HuggingFace `transformers.js` or standard HTTP clients.
*   🛡️ **TypeScript First:** Written in pure TypeScript with complete, autogenerated typings.

## Quickstart (React)

Track and load a model with progress UI using the built-in React hook:

```tsx
import React from 'react';
import { useModelLoader } from 'browser-llm-cache';

const MODEL_FILES = [
  'https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/onnx/model_quantized.onnx',
  'https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/tokenizer.json'
];

// Optional: Provide expected SHA-256 hashes to guarantee data integrity
const HASHES = {
  'https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/onnx/model_quantized.onnx': '44a86b1b...6f9c',
};

export function ModelLoaderComponent() {
  const {
    isReady,
    isDownloading,
    overallProgress,
    progress,
    error,
    startDownload,
    cancel,
    clear
  } = useModelLoader(MODEL_FILES, {
    hashes: HASHES,
    autoStart: false // Kick off manually
  });

  return (
    <div className="card">
      <h3>Model Asset Loader</h3>
      
      {!isDownloading && !isReady && (
        <button onClick={startDownload}>Download Model Assets (90MB)</button>
      )}

      {isDownloading && (
        <div>
          <p>Downloading... {overallProgress.percentage.toFixed(1)}%</p>
          <div className="progress-bar-container">
            <div 
              className="progress-bar" 
              style={{ width: `${overallProgress.percentage}%` }} 
            />
          </div>
          <p>Speed: {(overallProgress.speed / 1024 / 1024).toFixed(2)} MB/s</p>
          <button onClick={cancel}>Cancel Download</button>
        </div>
      )}

      {isReady && (
        <div>
          <p className="success">✅ All files loaded and verified!</p>
          <button onClick={clear}>Remove from Cache</button>
        </div>
      )}

      {error && <p className="error">Error: {error.message}</p>}
    </div>
  );
}
```

## HuggingFace / Transformers.js Integration

Directly override the default fetch engine of `@huggingface/transformers` to track downloads and secure assets:

```typescript
import { env, pipeline } from '@huggingface/transformers';
import { modelCache } from 'browser-llm-cache';

// 1. Redirect Transformers.js fetch call to our caching manager
env.fetch = modelCache.getCustomFetch({
  verifyOnLoad: false // Skips SHA-256 calculation for instant local loads
});

// 2. Subscribe to progress updates for the model files
const unsubscribe = modelCache.subscribe(
  'https://huggingface.co/Xenova/all-MiniLM-L6-v2/resolve/main/onnx/model_quantized.onnx',
  (progress) => {
    console.log(`Model Weight Download: ${progress.percentage.toFixed(1)}%`);
    console.log(`Current Speed: ${(progress.speed / 1024 / 1024).toFixed(2)} MB/s`);
  }
);

// 3. Run the pipeline as usual (will stream-cache to disk on first run)
const classifier = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2');

// Clean up listener when no longer needed
unsubscribe();
```

## Quickstart (Vanilla JavaScript)

```typescript
import { modelCache } from 'browser-llm-cache';

const fileUrl = 'https://example.com/models/llama-3.bin';
const expectedHash = 'e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855';

// Check if cached
const isCached = await modelCache.has(fileUrl, expectedHash);

if (!isCached) {
  const unsubscribe = modelCache.subscribe(fileUrl, (progress) => {
    console.log(`Download progress: ${progress.percentage.toFixed(2)}%`);
  });

  try {
    const response = await modelCache.retrieve(fileUrl, { sha256: expectedHash });
    console.log('Finished caching and verifying file!', response);
  } finally {
    unsubscribe();
  }
}
```

## API Reference

### `ModelCacheManager` (Shared Instance: `modelCache`)

#### `retrieve(url, options)`
Downloads a URL and stores it in the Cache Storage API if it is not already cached.
*   **Parameters:**
    *   `url`: `string` - The asset URL.
    *   `options`: `ModelCacheOptions` (optional)
        *   `sha256`: `string` - Expected SHA-256 checksum. If download doesn't match, throws error and deletes cache file.
        *   `verifyOnLoad`: `boolean` - If true, always reads cached file and recalculates SHA-256 rather than relying on IndexedDB metadata match.
        *   `headers`: `HeadersInit` - Custom HTTP headers for fetching.
        *   `signal`: `AbortSignal` - Abort signal to cancel download.
*   **Returns:** `Promise<Response>` - A fresh, readable Response stream matched from Cache Storage.

#### `has(url, expectedHash)`
Checks if an asset is cached.
*   **Parameters:**
    *   `url`: `string`
    *   `expectedHash`: `string` (optional) - If provided, checks if the cached asset hash matches. Bypasses file reading if hash matches IndexedDB metadata.
*   **Returns:** `Promise<boolean>`

#### `subscribe(url, callback)`
Subscribes to progress events for a URL.
*   **Parameters:**
    *   `url`: `string`
    *   `callback`: `(progress: DownloadProgress) => void`
        *   `DownloadProgress`: `{ loaded: number, total: number, percentage: number, speed: number }`
*   **Returns:** `() => void` - Unsubscribe function.

#### `delete(url)`
Removes the file from the cache and IndexedDB metadata store.
*   **Returns:** `Promise<boolean>`

#### `clear()`
Wipes the entire `browser-llm-cache` cache and clears IndexedDB metadata.
*   **Returns:** `Promise<void>`

#### `getCustomFetch(defaultOptions)`
Returns a fetch adapter function compatible with browser fetch standard.
